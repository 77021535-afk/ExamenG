Objetivo

Diseñar un sistema web para la gestión de inventario de una tienda (actualización de stock, eliminación de artículos y gestión de usuarios).

Herramientas Utilizadas

SQL Server → Base de datos relacional
Visual Studio 2022 → Desarrollo del software
GitHub → Publicación y control de versiones
IA (DeepSeek) → Generación de código inicial

Mapa Jerárquico de las Tablas

Orden de creación y dependencia de las tablas en la base de datos:

Cliente
Tabla raíz
Empleado
Tabla raíz
Proveedor
Tabla raíz
Categoria
Tabla raíz
VENTAS
Depende de Cliente y Empleado
Producto
Depende de Proveedor y Categoria
tblDetalleVenta
Depende de VENTAS y Producto

Interpretación:

  • Primero se crean Cliente, Empleado, Proveedor y Categoria (no tienen FKs)
  • Luego se crea Producto (depende de Proveedor y Categoria)
  • Después se crea VENTAS (depende de Cliente y Empleado)
  • Finalmente se crea tblDetalleVenta (depende de Producto y VENTAS)

Línea de Tiempo del Proyecto

Miércoles 18 Sept - Diseño Inicial

Planificación y diseño de la base de datos

Definición de requisitos, diseño del modelo entidad-relación y creación del diagrama de la base de datos.

Dificultad: Definir las relaciones correctas entre tablas

Jueves 19 Sept - Implementación BD

Creación de tablas y relaciones

Implementación física de la base de datos en SQL Server, creación de tablas, constraints y relaciones foreign key.

Dificultad: Problemas con la sintaxis de FOREIGN KEY constraints

Viernes 20 Sept - Desarrollo Backend

Conexión y clases de acceso a datos

Desarrollo de la capa de acceso a datos en C#, creación de la clase de conexión y métodos para CRUD operations.

Dificultad: Establecer la conexión correcta entre Visual Studio y SQL Server

Sábado 21 Sept - Interfaz de Usuario

Diseño de formularios y vistas

Creación de las interfaces de usuario en ASP.NET Web Forms, diseño de formularios para gestión de productos y usuarios.

Dificultad: Sincronización entre el equipo en el diseño de interfaces

Domingo 22 Sept - Integración

Integración frontend-backend

Conexión de la interfaz de usuario con la capa de acceso a datos, implementación de funcionalidades CRUD completas.

Dificultad: Depuración de errores de vinculación entre capas

Lunes 23 Sept - Pruebas y Depuración

Testing y corrección de errores

Pruebas integrales del sistema, detección y corrección de bugs, optimización de consultas SQL.

Dificultad: Resolver problemas de concurrencia en la base de datos

Martes 24 Sept - Documentación

Documentación y preparación para entrega

Elaboración de documentación técnica y manual de usuario, preparación de la presentación final.

Dificultad: Coordinar la finalización de todos los componentes a tiempo

Etapas del Desarrollo

1

Diseño de la base de datos – Creación de TIENDAUC con más de 5 tablas.

2

Uso de IA – Código inicial con DeepSeek, ajustado manualmente.

3

Implementación – Desarrollo en Visual Studio con conexión a SQL.

4

Publicación – Repositorio en GitHub con scripts y versiones del proyecto.

Conclusiones

  • Una base de datos bien diseñada asegura el funcionamiento del sistema.
  • Visual Studio 2022 facilitó la integración con SQL Server.
  • GitHub limita la subida de bases de datos completas.
  • El sistema cumplió objetivos, pero la publicación en SOMEE tuvo restricciones.

Recomendaciones

  • Homologar versiones de C# en el equipo.
  • Documentar la programación como algoritmo.
  • Explorar otras plataformas de publicación.
